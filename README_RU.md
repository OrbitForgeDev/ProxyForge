
#  🌊 ProxyForge


>**Асинхронный тестер прокси с поддержкой Telegram API и мультиязычным интерфейсом**

- Язык:[ENGLISH](README.md)/**RUSSIAN**

## 📒 Описание

`ProxyForge` — это мощный инструмент командной строки для массовой проверки прокси-серверов. Благодаря асинхронной архитектуре он проверяет до сотен прокси одновременно, автоматически сортирует их по типу (SOCKS5, SOCKS4, HTTPS, HTTP) и выделяет те, которые подходят для работы с Telegram.
## ✨ Функции

-   🚀 **Асинхронная проверка** — до 50+ прокси одновременно.
-   🧩 **Два режима работы**:
    -   `Стандартный` — общая проверка работоспособности.
    -   `Telegram` — проверка доступности Telegram API и ботов.
-   🗂 **Умная сортировка** — разделяет результаты по протоколам (SOCKS5 → SOCKS4 → HTTPS → HTTP).
-   🌐 **Мультиязычность** — поддерживает русский и английский языки (автоопределение системы).
-   🤖 **Интеграция с Telegram Bot API** — проверяет, может ли прокси работать с вашим конкретным ботом.
-   ⚙️ **Сохраняемые настройки** — конфигурация сохраняется между сессиями.
-   📊 **Прогресс-бар** — наглядный процесс проверки с `tqdm`.

## 🚀 Быстрый старт

Запуск главного меню:
```bash
python proxy_launcher.py
```

Интерактивное меню предложит выбрать режим, файл с прокси и дополнительные параметры.

### Пример работы в стандартном режиме

1.  Создай файл `proxies.txt` (по одной прокси на строку):

    *Файл proxies.txt приведён в качестве примера. Создайте его самостоятельно и добавьте свои прокси.*
    ```
    192.168.1.1:8080
    socks5://user:pass@192.168.1.2:1080
    https://192.168.1.3:3128
    ```
3.  Запусти `proxy_launcher.py`, выбери `1` → `Standard Proxy Check`.
4.  Получи результат: список живых прокси в `working_proxies.txt` с разделением по типам.

## 🛠 Использование в командной строке (CLI)

Ты можешь вызывать скрипты напрямую, минуя лаунчер.

### Стандартный тестер (`proxy_tester.py`)

```bash
python proxy_tester.py -i proxies.txt -o working.txt -t 10 -c 100 -l ru
```

**Ключи:**
-   `-i` — входной файл с прокси.
-   `-o` — выходной файл для рабочих прокси.
-   `-t` — таймаут в секундах (по умолч. 10).
-   `-c` — количество одновременных проверок (по умолч. 50).
-   `-l` — язык (`en` или `ru`).

### Тестер для Telegram (`proxy_tester_telegram.py`)

```bash
python proxy_tester_telegram.py -i proxies.txt -b "YOUR_BOT_TOKEN" -c 30
```

**Дополнительный ключ:**
-   `-b` — токен твоего Telegram бота для расширенной проверки (бот должен существовать).

## 🧠 Как это работает

1.  **Чтение прокси** — парсинг файла с поддержкой форматов `IP:PORT`, `http://`, `socks5://`.
2.  **Определение типа** — порт 1080/9050 → `SOCKS5`, иначе → `HTTP`.
3.  **Асинхронная проверка** — через `aiohttp` и `aiohttp_socks` отправляются запросы к тестовым URL (для Telegram — к `api.telegram.org`).
4.  **Фильтрация и сортировка** — сохраняются только отвечающие прокси, сгруппированные по протоколу.
5.  **Сохранение результатов** — в файл с заголовком и статистикой.

## ⚠️ Возможные ошибки и их решение

| Ошибка | Решение |
|--------|---------|
| `ModuleNotFoundError: No module named 'aiohttp'` | Установи зависимости: `pip install -r requirements.txt` |
| `❌ File proxies.txt not found!` | Создай файл с прокси или укажи правильный путь через `-i` |
| `ProxyConnectionError` | Прокси не отвечает — вероятно, мертв или требует авторизации. |
| `TimeoutError` | Увеличь таймаут ключом `-t 15` или `-t 20`. |
